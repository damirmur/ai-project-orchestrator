# AGENTS.md – Быстрое руководство

## Цель проекта
VK‑бот, который пересылает запросы пользователя в локальный LM‑Studio, позволяет модели редактировать файлы в `projects/` и применять изменения после подтверждения.

## Основные модули
- **src/main.ts** – стартует бота (`startVkBot()`), инициализирует persisted‑state.
- **src/modules/vk/vk.service.ts** – обработка сообщений, меню, выбор модели, чтение/запись файлов, автопинг.
- **src/modules/ai/lms.service.ts** – лёгкая обёртка над LM‑Studio API (`/v1/models`, `/v1/chat/completions`). Одна глобальная модель.
- **src/modules/project/opencode.service.ts** – безопасные чтение/запись файлов внутри `PROJECTS_ROOT`.
- **src/modules/state/state.service.ts** – **единый объект состояния**, сохраняет/восстанавливает `state.json`, управляет сессиями `userId:peerId`.

## Как работает состояние
1. При запуске `state.init()` читает `state.json` (если нет – создаёт пустой объект).
2. Для каждой комбинации `USER_ID:CHAT_PEER_ID` создаётся **сессия**:
   - `activeFilePath`
   - `stagedContent`
   - `lastAiResponse`
   - `pendingMessage`
   - `lastMessageId`
3. Любое изменение (выбор файла, генерация кода, подтверждение записи) сразу сохраняется (`state.save()`).
4. При перезапуске бот восстанавливает контекст из `state.json`.

## Команды бота (клавиатурные и текстовые)
- `🤖 Модели` / `/models` – выбрать модель LM‑Studio.
- `📂 Файлы` / `/list` – показать список файлов (10 штук).
- **Кнопка файла** (`auto_read`) – установить активный файл, загрузить его в контекст ИИ (без вывода кода).
- `📝 /write` (или `/write <path>`) – подготовить staged‑код из последнего ответа ИИ.
- `✅` / `❌` – подтвердить/отклонить запись в файл.
- `🏓 Пинг` – быстрый ответ «Понг!».  
- `/info`, `/status` – справка о системе и статусе LMS.

## Принцип работы диалога
1. Пользователь выбирает модель (если не выбрана – бот сохраняет запрос и просит выбрать модель).
2. Пользователь открывает файл → контент передаётся в `lmsService` как `fileContext`.
3. Пользователь задаёт запрос ИИ → получаем `session.lastAiResponse`.
4. Команда `/write` → код очищается от markdown, сохраняется в `session.stagedContent`.
5. Кнопка **✅** → `opencode.writeFile` сохраняет файл, после чего сессия очищается.

## Что удалено

- Файлы `analiz.md`, `architecture.md`, `todo.md` и тестовые скрипты в `projects/` – они не участвуют в работе бота.
- Любые «мертвые» участки кода в `vk.service.ts` (строки 135‑146).

## Как запустить в режиме watch
```bash
node --watch --experimental-transform-types src/main.ts
```
`node --watch` автоматически перезапустит процесс при каждом сохранении `.ts`‑файла; обработчик `SIGTERM` в `main.ts` корректно завершает VK‑сессию.
