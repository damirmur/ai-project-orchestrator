# AI Project Orchestrator

## Что это
VK‑бот с поддержкой нескольких AI провайдеров (LM‑Studio, OpenRouter). Позволяет выбирать модель, редактировать файлы в каталоге `projects/` и применять изменения после подтверждения.

## Быстрый старт
```bash
# Установить зависимости
npm ci

# Скопировать пример переменных окружения и заполнить
cp .env.example .env   # отредактировать .env

# Запустить в режиме watch (автоперезапуск при изменении кода)
npm run dev
# или: node --watch --env-file=.env --experimental-transform-types src/index.ts
```

## Переменные окружения (обязательно)
| Переменная | Описание |
|------------|----------|
| `VK_TOKEN` | Токен VK‑IO бота |
| `GROUP_ID` | ID группы бота |
| `USER_ID` | ID администратора, которому разрешено писать |
| `CHAT_PEER_ID` | ID чата, в котором бот слушает сообщения |
| `OPENROUTER_KEY` *(необязательно)* | API ключ OpenRouter для облачных моделей |
| `PROJECTS_ROOT` *(необязательно)* | Корень директории для файлов (по умолчанию `./projects`) |
| `LMS_URL` *(необязательно)* | URL LM‑Studio (по умолчанию `http://localhost:1234`) |

## Команды бота в VK
- `/start` – удалить старую клавиатуру, показать закреплённое меню (ℹ️ Инфо | 📱 Меню)
- ℹ️ Инфо – показать информацию о системе (стратегия, провайдеры, модель)
- 📱 Меню – показать inline меню с провайдерами и инструментами
- 🤖 Провайдеры – выбрать провайдера (lm-studio, cloud-openrouter) и стратегию
- 🤖 Модели – выбрать модель выбранного провайдера (с пагинацией)
- 📂 Файлы – показать список файлов проекта
- 📂 <имя файла> – выбрать файл для контекста
- `/write` – сохранить сгенерированный код в файл (после подтверждения)
- ✅ / ❌ – подтвердить или отклонить запись
- 🏓 Пинг – проверить что бот работает

## Провайдеры и модели
- **lm-studio** – локальные модели (список через `lms ls`)
- **cloud-openrouter** – облачные модели (бесплатные модели помечаются как "free", сортировка: free → популярные → остальные)

## RAG и эмбеддинги
Бот поддерживает создание RAG-проектов с использованием эмбеддингов LM Studio.

### Функции эмбеддингов
Доступны через `opencodeEmbedding` (экспорт `opencode`):
```typescript
// Получение эмбеддинга текста
const { vector } = await opencodeEmbedding.embedText("текст для эмбеддинга");
// vector: number[] — вектор размерности 768

// Конвертация для SQLite (Float32 → BLOB)
const blob = opencodeEmbedding.vectorToBlob(vector);
// blob: Uint8Array

// Обратная конвертация
const vec = opencodeEmbedding.blobToVector(blob);
// vec: number[]

// Cosine similarity
const score = opencodeEmbedding.cosine(vecA, vecB);
// score: number (0..1)
```

### Пример RAG-проекта
Проект `projects/rag-api/` — готовый пример:
- `src/ingest.ts` — индексация текстов с эмбеддингами
- `src/search.ts` — поиск по cosine similarity
- `GET /query?q=` — HTTP endpoint для поиска
- `scripts/seed.ts` — 18 чанков с документацией

## Как работает состояние
- При старте `state.service` читает `state.json` (если нет – создаёт пустой)
- Для каждого пользователя (`USER_ID:CHAT_PEER_ID`) – отдельная сессия
- `lastSelectedModelId` – последняя выбранная модель (приоритетная)
- Состояние сохраняется после каждого изменения
